SQL실습 

참고 	- 대부분이 대소문자 구분x 그러나 가독성을 위해 키워드는 대문자로 하는것도 좋다.
	- 선택이긴 하나 마지막에 ;을 붙여주는게 좋다고한다. (여러개의 SQL구문을 한번에 쓸 때 구분이 용이하다)
	- 줄넘김은 상관없다. 일자로 쭉 써도 되고 보기 좋게 줄 넘김해도 상관없다!

1. INSERT (삽입)
	INSERT INTO 테이블명 (컬럼명1,컬럼명2......) VALUES ('데이터1','데이터2'......) ;

1-1. 배치구문 (여러개의 데이터를 한 번에 입력할 때 유용)
	INSERT INTO 테이블명
		(컬럼명1,컬럼명2......)
	VALUES
		('데이터1','데이터2'......),
		('데이터1','데이터2'......),
		('데이터1','데이터2'......);
-> 주의점 : SQL은 트랜잭션을 걸어주지 않아도 이미 원자성을 가지고 있어 하나의 데이터만 잘못 입력되도 전체가 입력x



2. UPDATE (수정)
	UPDATE 테이블명 SET 컬럼명1 = '데이터1', 컬럼명2 = '데이터2'.... WHERE 컬럼명 = '데이터명';

-> 주의점 : WHERE은 조건설정으로 선택사항이나 설정하지 않으면 전체 데이터가 바뀜 설정하는 것이 대부분이다.



3. DELETE (삭제) 
	DELETE FROM 테이블명 WHERE 컬럼명 = 데이터 ;

-> 주의점 : 마찬가지로 WHERE은 조건설정으로 선택사항이나 설정하지 않으면 전체 데이터가 바뀜 주의 요망!


4. SELECT (검색)
	SELECT 컬럼명1,컬럼명2,컬럼명3.... 	FROM 테이블명;

	SELECT * FROM 테이블명; -> 모든 컬럼 읽기
	
	SELECT 컬럼명 FROM 테이블명 WHERE 컬럼명 = 데이터 ; -> 조건에 맞는 데이터 읽기

	SELECT 컬럼명 FROM 테이블명 WHERE 컬럼명 = 데이터 OREDER BY 정렬조건

정렬조건 
	ASC :오름차순 
	DESC :내림차순

5. SELECT JOIN (테이블끼리 병합 검색)

SELECT a.컬럼명1,a.컬럼명2,a.컬럼명3,a.컬럼명4,b.컬럼명1,b.컬럼명2... 
FROM A테이블명 a 
JOIN B테이블명 b ON a.외래키명 = b.키명 
WHERE 검색조건
ORDER BY 정렬조건

-> 주의점 : 여기서 a,b는 알리어스 이며 선택사항이나 만약 두 테이블에 같은 컬럼명이 존재할 경우 에러가 날 수 있으므로
하는 것이 좋은 것 같고 알리어스명은 임의로 a,b로 한 것이므로 자유롭게 변경가능 ON 이후에는 두 테이블을 연결해줄 공통 컬럼이 있어야함.
